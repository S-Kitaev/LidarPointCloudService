# Технические требования
## Windows

- ### Необходимое ПО
  - Docker Desktop (установлен и запущен).
  - WSL 2 (Windows Subsystem for Linux) — включён и настроен.
- ### Свободное место
    - ≥ 2 ГБ свободного дискового пространства для образов/контейнеров.
- ### Сеть

    - Доступ в Интернет (для загрузки образов).

- ### Проверка / включение WSL 2 (если WSL выключен):

```Bash
# Запустить PowerShell от имени администратора и выполнить:
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# Перезагрузите систему, затем в PowerShell:
wsl --set-default-version 2

# Установите дистрибутив (пример):
wsl --install -d Ubuntu

# Проверить состояние WSL:
wsl -l -v
```

 - Примечание: после установки WSL убедитесь, что Docker Desktop использует WSL 2 backend.

## Linux

- ### Необходимое ПО

    - Docker Engine (установлен и запущен).
    - Docker Compose (или плагин ```docker compose```) установлен.
    - Все остальные Docker-компоненты, требуемые для запуска образов (containerd, runc и т.п.), должны быть установлены вместе с Docker.

- ### Свободное место

    - ≥ 2 ГБ свободного дискового пространства для образов/контейнеров.

- ### Сеть

    - Возможность подключения к Wi-Fi / к локальной сети и доступ в Интернет (для клонирования репозитория и загрузки образов).

- ### Проверки (команды)
```bash
# Проверить Docker
docker --version
docker info

# Проверить Docker Compose
docker compose version   # или: docker-compose --version
```
