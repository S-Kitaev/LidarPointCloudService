## **Календарный план (на 4 месяца) Фаза 2**

| **Период**  | **Основные задачи и контрольные точки**                                                                                                           |
|-------------|---------------------------------------------------------------------------------------------------------------------------------------------------|
| **Месяц 1** | **Подготовка и анализ**                                                                                                                           |
| Неделя 1    | \- Утверждение ТЗ и концепции проекта на вторую фазу, утверждение задач                                                                           |
| Неделя 2    | \- Рефакторинг рабочих частей приложения                                                                                                          |
| Неделя 3    | \- Подготовка первой версии ПМИ                                                                                                                   |
| Неделя 4    | \- Разработка первичных макетов страницы capture                                                                                                  |
| **Месяц 2** | **Разработка прототипов и начальная реализация**                                                                                                  |
| Неделя 5    | \- Формирование облачной БД (проект БД, подключение к ЦХД); конфигурация окружения, .env                                                          |
| Неделя 6    | \- Начало разработки веб-интерфейса (frontend). - Старт разработки API (backend)                                                                  |
| Неделя 7    | \- Разработка скриптов работы с LiDAR: запись данных, управление шаговым двигателем, создание ```scan.py```, ```lidar.py```, ```engine.py```      |
| Неделя 8    | \- Интеграция скриптов с бэкендом: вызов команд по SSH/paramiko, первые тесты передачи данных и сохранения результатов                            |
| **Месяц 3** | **Интеграция и тестирование**                                                                                                                     |
| Неделя 9    | \- Доработка визуализации облака точек правильная настройка осей координат и трансформации данный                                                 |
| Неделя 10   | \- Автоматизация запуска съёмки, WebSocket логов, потоковое отображение stdout в веб-логе.                                                        |
| Неделя 11   | \- Создание страницы съёмки (capture): верстка, аккордеоны, логи и управление; реализация кнопок (ping, connect, start/stop/download)             |
| Неделя 12   | \- Создание страницы подключения к ЦХД и синхронизации облаков; интеграционные тесты (SFTP, скачивание файлов, сохранение в scans/)               |
| **Месяц 4** | **Пилотный запуск, доработка и подготовка к защите**                                                                                              |
| Неделя 13   | \- Пилотный запуск: демонстрация рабочей версии (запуск съёмки, поток логов, скачивание результатов)                                              |
| Неделя 14   | \- Сбор обратной связи от тестирующих: фиксы UI/UX, исправление ошибок подключения и сетевых нюансов (VPN/прокси, Docker networking)              |
| Неделя 15   | \- Финальная доработка функционала: обработка краевых случаев, повторные запуски сканирования, надежное управление задачами и кнопками интерфейса |
| Неделя 16   | \- Подготовка финальной документации, README и руководства пользователя; подготовка презентации и материалов для защиты проекта                   |
