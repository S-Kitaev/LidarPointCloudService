# Концептуальный проект ЛИДАР

## 1. Цели и задачи проекта

- Создание низкобюджетного функционального инструментария для трёхмерного лазерного сканирования с использованием LiDAR.  
- Обзор инструментов получения облака точек (карты) и алгоритмов по преобразованию полученных данных в геометрические объекты.  
- Реализация аппаратно-программного комплекса.  

---

## 2. Требования и ограничения

- Датчик LiDAR с дальностью измерений *N* м.  
- Одноплатный компьютер Raspberry Pi 3 Model B+.  
- Встроенный в плату Wi-Fi модуль для передачи данных.  
- Компьютер с ОС Windows и необходимыми вычислительными ресурсами для обработки данных.  
- Программное обеспечение для визуализации карты помещения (объекта).  

---

## 3. Описание системы

- Установка состоит из следующих элементов: **двигатель**, **драйвер**, **адаптер**, **микрокомпьютер** и **лидар**.  
- Двигатель отвечает за поворотную функцию установки.  
- ЛИДАР снимает показания расстояний до объекта.  
- Адаптер выступает промежуточным звеном для подключения драйвера к сети.  
- Микрокомпьютер собирает данные с ЛИДАРа и даёт сигнал двигателю.  

![i8.jpg](images%2Fi8.jpg)

Лидар установлен на статичную платформу с подвижной головкой.  

Платформа размещается оператором в помещении, собирая данные о расстояниях до объектов.  

Полученные данные передаются на компьютер в виде **txt-файлов** посредством **Wi-Fi-соединения через VNC Viewer**, который обеспечивает доступ к Raspberry с компьютера или ноутбука.  

На Raspberry предварительно должен быть установлен и запущен **VNC Server**.  

После сканирования точек (при помощи Python-скрипта) показатели записываются в файлы на микрокомпьютер.  

Данные отправляются в локальное/облачное хранилище измерительного комплекса, а также в централизованное локальное/облачное хранилище.  

С помощью ПО на компьютере данные предобрабатываются и визуализируются пользователю в виде карты окружающей среды.

## 4. Идентификация рисков

- Помехи в сети Wi-Fi могут вызвать потерю данных.  
- Проблемы с качеством сигнала ЛИДАР-датчика в условиях полупрозрачных объектов.  
- Перегрев компонент системы при продолжительной работе.  
- Потеря достоверности данных при передаче.  

---

## 5. Выбор стратегии реализации

- Использование стандартных протоколов Wi-Fi для передачи данных.  
- Разработка программного обеспечения для обработки данных (преобразования координат из полярной системы в декартову).  
- Использование программы для визуализации данных на компьютере.
